# 仮想通貨自動売買システム「一攫千金くん」要件定義書 v0.1

## 1. 目的

本システムは、仮想通貨の価格情報と売買シグナルをもとに自動売買を行う。購入時・売却時にはLINE通知を行い、売却時には損益や総資産などを通知する。

初期版では、実資金をいきなり動かさず、価格取得、DB保存、シグナル判定、仮想注文、LINE通知までを優先する。実注文APIは十分な検証後に実装する。

## 2. システム名

仮想通貨自動売買システム「一攫千金くん」

## 3. 開発方針

- 最初は本番運用を見据えて作る。
- ただし実注文は最後にする。
- まずはペーパートレードで検証する。
- 誤発注、二重注文、損切り漏れ、APIエラー時の暴走を最優先で防ぐ。
- 利益最大化を目指すが、最低限の安全停止条件は必ず実装する。

## 4. 対象範囲

### 4.1 初期スコープ

- Coincheck Public APIによる価格取得
- Microsoft SQL Serverへの価格・ログ保存
- ローソク足生成
- SMA / RSI / MACDなどの指標計算
- 売買シグナル判定
- 仮想注文のDB保存
- LINE通知
- リスク停止判定

### 4.2 後続スコープ

- Coincheck Private APIによる残高取得
- 注文状態確認
- 実注文API
- 約定確認
- GUI
- 対象通貨拡張
- 売買ロジック評価とパラメータ調整

## 5. 対象外

初期版では以下を対象外とする。

- 実資金による自動売買
- レバレッジ取引
- 複数取引所対応
- 高度なAI売買判断
- 完全自動のパラメータ最適化
- Web画面、スマホアプリ

## 6. 前提条件

| 項目 | 内容 |
|---|---|
| 取引所 | Coincheck |
| API種別 | Public APIから開始。Private APIは後続 |
| 取引種別 | 現物のみ |
| 取引形式 | 取引所取引を前提 |
| 初期対象通貨 | BTC/JPY |
| 将来対象通貨 | ETHなどへ拡張予定 |
| エントリー | 成行買い想定 |
| 利確 | 指値売り想定 |
| 損切り | 成行売り想定 |

| DB | Microsoft SQL Server |

| DBサーバー | MSI\SQLEXPRESS |
| DB名 | CryptoBot |
| 言語 | Python |

## 7. 機能要件

### 7.1 価格取得

- Coincheck Public APIから現在価格を取得する。
- 取得対象はBTC/JPYを初期値とする。
- 取得した価格、bid、ask、volume、取得日時をDBに保存する。

### 7.2 ローソク足管理

- market_pricesのデータをもとにローソク足を生成する。
- 初期対象は5分足、15分足とする。
- 将来的に1分足、1時間足にも対応する。

### 7.3 シグナル判定

- SMA、RSI、MACDなどを用いて売買判断の材料を作る。
- 複数時間足の方向が一致したときに買い候補とする。
- RSIが過熱している場合は買いを見送る。

### 7.4 注文管理

初期版では実注文ではなく、仮想注文としてDBに保存する。

後続で以下を実装する。

- 成行買い
- 指値売り
- 成行売り
- 注文キャンセル
- 約定確認
- 二重注文防止

### 7.5 ポジション管理

- 現在保有中のポジションがあるか判定する。
- エントリー価格、数量、利確価格、損切り価格を保存する。
- 決済後はポジションをクローズする。

### 7.6 損益管理

- 売却時に粗利、手数料、純利益、利益率を計算する。
- trade履歴としてDBに保存する。

### 7.7 LINE通知

通知対象は以下とする。

- システム起動
- システム停止
- 価格取得エラー
- 買い注文発生
- 売り注文発生
- 損切り発生
- 利確発生
- 緊急停止

### 7.8 リスク管理

- DB上の自動売買無効フラグを見て停止する。
- emergency_stopが有効なら停止する。
- 将来的に日次最大損失、連続損切り、急変動停止を実装する。

## 8. 非機能要件

| 項目 | 内容 |
|---|---|
| 安全性 | 実注文前に必ずペーパートレードを行う |
| ログ | エラー、取引、通知、リスクイベントをDBまたはログに残す |
| 再現性 | requirements.txtと.env.exampleで環境再構築できること |
| 保守性 | app配下を機能別に分割する |
| 拡張性 | 対象通貨、指標、取引所を追加できる構成にする |
| 秘密情報 | .envに置き、GitHubにはコミットしない |

## 9. 初期パラメータ

| 項目 | 値 |
|---|---:|
| 注文金額 | 5,000円 |
| 利確率 | 0.5% |
| 損切り率 | 0.7% |
| メインループ間隔 | 60秒 |
| 1日最大損失率 | 3% |
| 連続損切り停止回数 | 3回 |

## 10. 完了条件

初期版の完了条件は以下とする。

- main.pyが起動する。
- DB接続できる。
- bot_statusを読み取れる。
- 自動売買OFF時に処理停止できる。
- Coincheck Public APIから価格取得できる。
- market_pricesへ価格をINSERTできる。
- 仮想注文をordersへ保存できる。
- LINE通知できる。
- pytestが通る。
